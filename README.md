# 🦖 Lógica de Programação: Lista 4 - Operadores Lógicos e Decisões 🖥️

![Status](https://img.shields.io/badge/Status-Conclu%C3%ADdo-brightgreen?style=for-the-badge)

Esta é a quarta atividade de Lógica de Programação do curso **Técnico em Desenvolvimento de Sistemas**. O foco central desta lista foi o domínio dos **Operadores Lógicos** para a construção de decisões complexas.

O desafio consistiu em criar algoritmos que não dependem apenas de condições simples, mas de combinações lógicas para validar processos como aprovações de crédito, triagens de currículos e regras de descontos.

---

## 📂 Projetos Realizados

| Algoritmo | Descrição Técnica | Status |
| :--- | :--- | :--- |
| `Calculadora de desconto` | Aplicação de descontos utilizando operadores para verificar status VIP, preço mínimo e pendências financeiras simultaneamente. | ✅ |
| `Análise de currículo` | Triagem técnica que utiliza o operador `OU` para aceitar diferentes linguagens (Python/JS) e o operador `E` para validar a maioridade. | ✅ |
| `Avaliação de empréstimo` | Sistema de análise de risco que cruza renda ou fiador com a obrigatoriedade de nome limpo (operador `E`). | ✅ |
| `Ingresso gratuito` | Validação de faixas etárias utilizando lógica de intervalos para garantir gratuidade a crianças e idosos. | ✅ |
| `Nota e frequência` | Sistema acadêmico que exige o cumprimento simultâneo de média e presença mínima para aprovação. | ✅ |
| `Checagem de estoque` | Verificação de disponibilidade com tratamento de erros para entradas de dados fora do padrão esperado. | ✅ |

---

## ⚠️ Regras Importantes 

> **O Poder dos Operadores:** O foco desta lista foi entender como os operadores `E` e `OU` mudam completamente o fluxo do programa.
> **Lógica Robusta:** Além de resolver o problema, os códigos foram pensados para evitar erros de digitação e garantir que todas as condições de negócio fossem atendidas.
